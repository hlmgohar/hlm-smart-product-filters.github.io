01 Getting Started

Install, activate, and find your way around the plugin.

After installing and activating the plugin, a new HLM Product Filters menu item appears in your WordPress admin sidebar. It contains two pages:

  • Settings — Global options that apply to all filters across your store.
  • Filter Builder — Where you create, configure, and manage individual filters.
💡
Quick Start: The fastest way to get up and running is to load a Quick Start Template from the Filter Builder page. See Building Filters below.

03 Building Filters

Navigate to HLM Product Filters › Filter Builder in your admin menu.

Using a Quick Start Template

The easiest way to get started:

  1. At the top of the Filter Builder page, locate the Quick Start Templates section.
  2. Click on a template card (e.g., Comprehensive Shop).
  3. A ready-to-use set of filters is created for your store.
  4. Click Save Filters to keep your configuration.
⚠️
Heads up: Loading a template replaces any existing filters. Export your current setup first if you want a backup.

Creating a Filter from Scratch

  1. Click the Add Filter button.
  2. Fill in the settings across the available tabs (detailed below).
  3. Click Save Filters when finished.

Managing Filters

  • Reorder — Drag the handle icon on the left to rearrange display order.
  • Expand / Collapse — Click a filter header to toggle its settings.
  • Expand All / Collapse All — Manage all panels at once.
  • Remove — Click the red "Remove" link to delete a filter.

Filter Settings Tabs

Each filter has five configuration tabs:

General Tab

FieldDescription
LabelThe name customers see (e.g., "Color," "Size," "Price Range"). Required.
SourceWhere the filter gets its data: product categories, tags, attributes (Color, Size, etc.), or meta fields (_price).

Common source choices: Product categories, Product tags, Color / Size / Brand (any attribute), or Meta: _price (for price-based filtering).

Behavior Tab

FieldDescription
Allow multi-selectLets customers select more than one option at a time.
OperatorOR = show products matching any selection. AND = show products matching all selections.

UI Tab

FieldDescription
Display typeHow the filter looks. See Filter Types Explained.
List layoutFor checkbox/swatch filters: Stacked or Inline.
Swatch typeFor swatch filters only: Color, Image URL, or Text.
Show more thresholdHide options after this many items with a "Show more" button. 0 = show all.
Slider stepIncrement between values (e.g., 1 for whole numbers, 0.01 for decimals).
Range prefixSymbol before the value (e.g., "$").
Range suffixSymbol after the value (e.g., "kg").

Controls where the filter appears. See Controlling Visibility.

Advanced Tab

KeyURL parameter name. Keep it short (e.g., "color," "size"). Auto-generated.
💡
You usually don't need to change the Advanced settings unless you have a specific technical requirement.

04 Filter Types Explained

Choose the right display type for each filter.

☑

Checkboxes (List)

The most common filter type. Shows a list of options with checkboxes.

Best for: Categories, tags, brands, multi-option attributes.
▾

Dropdown

A compact select menu. Customers click to expand and choose.

Best for: Filters with many options where space is limited.
◆

Swatch

Visual swatches: color squares, image thumbnails, or text chips.

Best for: Colors, textures, sizes (S/M/L/XL).
#

Range (Min/Max)

Two number input fields for entering a minimum and maximum value.

Best for: Price filtering or any numeric attribute.
⟷

Slider

An interactive slider with two handles for selecting a range visually.

Best for: Price with a touch-friendly interface.
ℹ️
Note: Range and Slider filters require a meta field data source (e.g., _price) to function.

05 Working with Swatches

Display visual elements instead of plain text labels.

Color Swatches

Displays colored squares for each filter option.

  1. Set the filter Display type to Swatch.
  2. Set the Swatch type to Color.
  3. Click the Swatches button in the filter header to open the Swatch Editor.
  4. For each term, use the color picker or type a hex code (e.g., #ff0000 for red).
  5. Click Save in the Swatch Editor.

Swatch Editor features:

  • Search box — Type to find a specific term. Matches are highlighted and sorted to the top.
  • Populate Predefined Colors — Auto-fills colors from a built-in database, matching by name (e.g., "Red" → #ff0000).

Image Swatches

Displays small image thumbnails for each option.

  1. Set Display type to Swatch.
  2. Set Swatch type to Image URL.
  3. Open the Swatch Editor.
  4. Paste an image URL for each term (e.g., https://yoursite.com/wp-content/uploads/texture.jpg).
  5. Click Save.

Text Swatches

Displays text labels as styled chips — the simplest swatch type.

  1. Set Display type to Swatch.
  2. Set Swatch type to Text.
  3. That's it! No Swatch Editor needed.
💡
Text swatches automatically use the term name as the label. If your Size attribute has "S," "M," "L," and "XL," these appear as styled chips on the frontend.

06 Controlling Where Filters Appear

The Visibility tab on each filter controls exactly where it shows up.

Hide Empty Terms

Enable this to hide filter options with no matching products. If no products are tagged "Vintage," that option won't appear.

Category Visibility

Control which product category pages display this filter:

All

Shows on every page. This is the default.

Include

Shows only on categories you select.

Exclude

Shows everywhere except categories you select.

When using Include or Exclude mode, a category list appears for selection. Enable Include child categories to extend the rule to all subcategories.

Tag Visibility

Works identically to Category Visibility, but applies to product tag pages.

07 Excluding Specific Terms

Prevent specific terms from appearing in a filter.

For example, if you have a "Color" filter but never want "Discontinued Gray" to show up:

  1. Open the filter's Visibility tab.
  2. Find the Exclude specific terms section.
  3. The term list loads when you click the Visibility tab.
  4. Click individual terms to select them (click again to deselect).
  5. Use Select all or Clear buttons for quick actions.
  6. Click Save Filters.

Excluded terms will no longer appear in that filter on the frontend.

08 Adding Filters to Your Pages

Two methods to display filters on your store.

Method 1: Shortcode (Recommended)

Place this shortcode anywhere you want filters to appear:

[hlm_smart_product_filters]

Works in pages (WordPress editor), widget areas, theme template files, or page builder elements like Elementor and WPBakery.

Method 2: Auto Injection

Automatically add filters without editing templates:

  1. Go to Settings › General Tab.
  2. Set Render mode to "Auto inject only" or "Shortcode + auto inject."
  3. Enable the relevant auto inject checkboxes (Shop, Category archives, Tag archives).
  4. Save your settings.

How Filtering Works for Customers

  1. Customers see the filter panel on the shop page.
  2. They select options — check boxes, pick colors, slide ranges, etc.
  3. The page updates to show only matching products.
  4. Active filters are visually highlighted so customers know what's applied.
  5. A Clear all link removes all active filters at once.
💡
If Sort By is enabled, customers can also reorder results by price, popularity, rating, newest, or title.

09 Import & Export

Navigate to HLM Product Filters › Import / Export.

Exporting

  1. Click Download as JSON to save a backup file.
  2. Or click Copy to Clipboard to copy the configuration text.

This exports all settings and filter definitions as a single JSON file.

Importing

  1. Click Choose File and select a previously exported .json file.
  2. Or paste JSON text directly into the text area.
  3. Click the import button.
  4. Confirm when prompted.
⚠️
Warning: Importing replaces all current settings and filters. Always export first as a backup.
💡
Click Download Sample JSON to see the expected format — useful for understanding the structure or manually editing configuration files.

10 Troubleshooting

Common issues and how to resolve them.

Filters Are Not Showing on My Shop Page

  1. Verify you've created at least one filter in the Filter Builder.
  2. Check Render mode in Settings — should be "Shortcode only" or "Shortcode + auto inject."
  3. If using a shortcode, confirm [hlm_smart_product_filters] is on the correct page.
  4. If using auto inject, verify the relevant "Auto inject on…" checkboxes are enabled.

Some Filter Options Are Missing

  • Hide empty terms enabled? Options with no products are hidden automatically.
  • Exclude specific terms — Verify the missing options haven't been manually excluded.
  • Show more threshold — Options beyond this number are hidden behind a "Show more" button.

Swatch Colors Are Not Showing

  1. Confirm Display type is Swatch and Swatch type is Color.
  2. Open the Swatch Editor via the "Swatches" button.
  3. Ensure each term has a color assigned.
  4. Try Populate Predefined Colors to auto-fill common names.
  5. Save in the Swatch Editor, then Save Filters on the main page.

Filters Are Slow to Load

  1. Turn on Enable caching in Settings.
  2. Increase Cache TTL if products don't change often (e.g., 3600 for 1 hour).

Products Are Not Filtering Correctly

  1. Verify Source is set to the correct attribute or taxonomy.
  2. Check Operator: OR = any match, AND = all must match.
  3. Confirm Allow multi-select is enabled if needed.
  4. Enable Debug mode in Settings to log operations for investigation.

Excluded Terms Still Appear

  1. Ensure you clicked Save Filters after selecting terms to exclude.
  2. Reload and check the Visibility tab — excluded terms should appear as selected.
  3. Clear your browser cache and any site caching plugins.
